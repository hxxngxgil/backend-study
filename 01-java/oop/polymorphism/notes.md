# Polymorphism (다형성)

## 다형성이란
다형성은 하나의 객체가 여러 타입으로 동작할 수 있는 기능이다.

즉, 부모 타입으로 자식 객체를 다룰 수 있다.

---

## 다형성 시작

예시

class Animal {}
class Dog extends Animal {}

Animal animal = new Dog();

→ 부모 타입으로 자식 객체를 참조할 수 있다.

---

## 다형성과 캐스팅

부모 타입으로 자식 객체를 참조하면  
자식의 기능은 바로 사용할 수 없다.

→ 필요 시 캐스팅을 사용한다.

---

## 캐스팅 종류

### 업캐스팅 (Upcasting)
- 자식 → 부모
- 자동 형변환
- 안전함

예시

Animal animal = new Dog();

---

### 다운캐스팅 (Downcasting)
- 부모 → 자식
- 명시적 형변환 필요
- 위험할 수 있음

예시

Dog dog = (Dog) animal;

---

## 다운캐스팅 주의점

실제 객체 타입이 자식이 아닐 경우  
오류가 발생한다.

---

## instanceof

객체의 실제 타입을 확인할 때 사용한다.

예시

if (animal instanceof Dog) {
    Dog dog = (Dog) animal;
}

---

## 다형성과 메서드 오버라이딩

오버라이딩된 메서드는  
참조 타입이 아니라 실제 객체 기준으로 실행된다.

예시

Animal animal = new Dog();
animal.move();  // Dog의 메서드 실행

---

## 다형성 활용

- 코드 유연성 증가
- 확장성 향상
- 유지보수 용이

---

## 추상 클래스 (abstract class)

객체를 직접 생성할 수 없는 클래스이다.  
공통 기능을 정의하고 상속을 통해 사용한다.

예시

abstract class Animal {
    abstract void sound();
}

---

## 인터페이스 (interface)

모든 메서드가 추상 메서드인 구조이다.  
구현을 강제하기 위해 사용한다.

예시

interface Animal {
    void sound();
}

---

## 인터페이스 다중 구현

인터페이스는 여러 개 구현이 가능하다.

예시

class Dog implements Animal, Pet {
}

---

## 클래스 vs 인터페이스

- 클래스 → 기능 + 상태 포함
- 인터페이스 → 기능 정의 (구현 없음)

---

## 다형성의 역할과 구현

- 역할 (인터페이스)
- 구현 (클래스)

→ 역할과 구현을 분리하면 유연한 설계 가능

---

## OCP (Open-Closed Principle)

확장에는 열려 있고, 변경에는 닫혀 있어야 한다.

즉
- 기존 코드를 수정하지 않고
- 새로운 기능을 추가할 수 있어야 한다.

---

## 핵심 정리

1. 다형성은 하나의 타입으로 여러 객체를 다루는 것이다.
2. 업캐스팅은 자동, 다운캐스팅은 위험하다.
3. 오버라이딩은 실제 객체 기준으로 실행된다.
4. 인터페이스는 다중 구현이 가능하다.
5. 역할과 구현을 분리하면 설계가 유연해진다.
6. OCP는 변경 없이 확장이 가능해야 한다.
