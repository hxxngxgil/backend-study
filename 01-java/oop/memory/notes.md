# Memory Structure (자바 메모리 구조)

## 자바 메모리 구조

자바의 메모리는 크게 3가지 영역으로 나뉜다.

- 메서드 영역 (Method Area)
- 스택 영역 (Stack Area)
- 힙 영역 (Heap Area)

---

## 메서드 영역 (Method Area)

클래스 정보와 메서드 정보를 저장하는 영역이다.

저장되는 내용
- 클래스 정보
- 메서드 정보
- static 변수

프로그램 실행 시 한 번만 생성되고 공통으로 사용된다.

---

## 스택 영역 (Stack Area)

메서드 호출 시 생성되는 영역이다.

저장되는 내용
- 지역 변수
- 매개변수
- 메서드 호출 정보

특징
- 메서드 호출 시 생성
- 메서드 종료 시 자동 제거

---

## 힙 영역 (Heap Area)

객체가 생성되는 영역이다.

저장되는 내용
- 객체 (new로 생성된 데이터)

특징
- 참조형 데이터 저장
- 여러 곳에서 공유 가능

---

## 스택과 힙 관계

- 스택에는 변수와 참조값이 저장된다.
- 힙에는 실제 객체 데이터가 저장된다.

예시

Student student = new Student();

→ student (참조값) → 스택  
→ 실제 객체 → 힙

---

## 스택 구조 (Stack)

스택은 후입선출(LIFO, Last In First Out) 구조이다.

- 나중에 들어온 데이터가 먼저 나간다.

---

## 큐 구조 (Queue)

큐는 선입선출(FIFO, First In First Out) 구조이다.

- 먼저 들어온 데이터가 먼저 나간다.

---

## 스택 영역 특징

- 메서드 호출 순서대로 쌓인다.
- 마지막에 호출된 메서드가 먼저 종료된다.
- LIFO 구조

---

## 힙 영역 특징

- 객체가 저장되는 공간
- 참조값을 통해 접근
- 여러 변수에서 하나의 객체를 참조 가능

---

## 가비지 컬렉션 (GC)

가비지 컬렉션은 더 이상 사용되지 않는 객체를 자동으로 제거하는 기능이다.

---

## GC 동작 원리

어떤 객체도 참조하지 않는 상태가 되면  
해당 객체는 가비지 컬렉션 대상이 된다.

예시

Student student = new Student();
student = null;

→ 기존 객체는 더 이상 참조되지 않음  
→ GC 대상이 됨

---

## GC 특징

- 개발자가 직접 메모리를 해제하지 않아도 된다.
- 힙 영역의 메모리를 자동으로 관리한다.
- 사용하지 않는 객체를 제거하여 메모리를 효율적으로 사용한다.

---

## 핵심 정리

1. 자바 메모리는 메서드, 스택, 힙 영역으로 나뉜다.
2. 스택은 변수와 메서드 호출 정보를 저장한다.
3. 힙은 객체를 저장한다.
4. 스택은 LIFO, 큐는 FIFO 구조이다.
5. 참조형 변수는 스택에, 객체는 힙에 저장된다.
6. 참조되지 않는 객체는 GC에 의해 자동으로 제거된다.
