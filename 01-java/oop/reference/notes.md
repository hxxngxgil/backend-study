# Primitive Type & Reference Type (기본형과 참조형)

## 기본형과 참조형

자바의 변수 타입은 크게 기본형과 참조형으로 나눌 수 있다.

### 기본형
기본형은 변수에 값을 직접 저장한다.

예시  
int, long, double, boolean, char

### 참조형
참조형은 객체의 위치(참조값, 주소값)를 저장한다.

예시  
클래스, 배열

기본형을 제외한 나머지 변수는 모두 참조형이다.

---

## 기본형 vs 참조형 차이

### 기본형
- 값을 직접 저장한다.
- 산술 연산이 가능하다.
- null을 할당할 수 없다.

### 참조형
- 객체의 주소(참조값)를 저장한다.
- 산술 연산을 할 수 없다.
- null을 할당할 수 있다.

---

## 대입

### 기본형 대입
기본형은 값을 그대로 복사해서 전달한다.

예시

int a = 10;
int b = a;

→ b에는 10이라는 값이 복사된다.

### 참조형 대입
참조형은 참조값을 복사해서 전달한다.

예시

Student s1 = new Student();
Student s2 = s1;

→ s2에는 객체 자체가 아니라 참조값이 복사된다.

---

## 메서드 호출

### 기본형
메서드 호출 시 값을 복사해서 전달한다.  
따라서 메서드 안에서 값을 변경해도 호출자의 변수에는 영향이 없다.

### 참조형
메서드 호출 시 참조값을 복사해서 전달한다.  
따라서 전달된 객체의 멤버 변수를 변경하면 호출자의 객체도 변경될 수 있다.

---

## 변수와 초기화

변수는 사용하기 전에 초기화가 필요하다.

### 지역 변수
지역 변수는 직접 초기화해야 한다.

예시

int a;
→ 초기화하지 않으면 사용할 수 없다.

### 멤버 변수
멤버 변수는 자동으로 초기화된다.

초기값 예시
- 숫자형 → 0
- boolean → false
- 참조형 → null

---

## null

null은 참조형 변수가 아직 어떤 객체도 가리키지 않는다는 뜻이다.

예시

Student student = null;

→ student는 참조할 객체가 없는 상태이다.

---

## NullPointerException

참조값이 null인 변수를 사용하면 NullPointerException이 발생한다.

예시

Student student = null;
student.name = "Tom";

→ null은 참조할 대상이 없기 때문에 예외가 발생한다.

---

## 핵심 정리

1. 기본형은 값을 직접 저장한다.
2. 참조형은 참조값(주소값)을 저장한다.
3. 기본형은 값을 복사하고, 참조형은 참조값을 복사한다.
4. 참조형은 null을 가질 수 있다.
5. null 상태에서 객체를 사용하면 NullPointerException이 발생한다.
