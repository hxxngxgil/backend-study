# Access Modifier & Encapsulation (접근 제어자와 캡슐화)

## 접근 제어자란
접근 제어자는 클래스, 변수, 메서드에 대한 접근 범위를 제한하는 키워드이다.  
외부에서 직접 접근하는 것을 막아 안정적인 코드를 만들기 위해 사용한다.

---

## 접근 제어자를 사용하는 이유

객체의 내부 데이터를 보호하기 위해 사용한다.

예시 (스피커)

스피커는 음량을 100까지만 올려야 정상 동작한다.  
하지만 외부에서 직접 데이터에 접근하면 100 이상의 값을 넣을 수 있다.

→ 접근 제어자를 사용하면 이런 문제를 사전에 방지할 수 있다.

---

## 접근 제어자 종류

### private
- 모든 외부 접근을 차단한다.
- 같은 클래스 내부에서만 사용 가능

---

### default (package-private)
- 같은 패키지 내부에서만 접근 가능
- 접근 제어자를 명시하지 않으면 default가 적용된다.

---

### protected
- 같은 패키지에서 접근 가능
- 다른 패키지라도 상속 관계라면 접근 가능

---

### public
- 모든 외부에서 접근 가능

---

## 접근 범위 정리

private → default → protected → public  

→ 왼쪽으로 갈수록 접근 제한이 강하다.

---

## 클래스 레벨 접근 제어자

클래스에는 다음 두 가지만 사용할 수 있다.

- public
- default

---

## public 클래스 규칙

- public 클래스는 파일명과 반드시 같아야 한다.
- 하나의 파일에는 public 클래스는 하나만 존재할 수 있다.

---

## default 클래스

- 하나의 파일에 여러 개 정의 가능하다.

---

## 캡슐화 (Encapsulation)

캡슐화는 데이터와 기능을 하나로 묶고  
외부에서 직접 접근하지 못하도록 보호하는 것이다.

---

## 캡슐화 핵심

1. 데이터는 숨긴다 (private 사용)
2. 기능은 필요한 부분만 외부에 노출한다

→ 즉, 직접 접근은 막고 메서드를 통해서만 제어한다.

---

## 핵심 정리

1. 접근 제어자는 외부 접근을 제한하기 위해 사용한다.
2. private이 가장 강하고 public이 가장 약하다.
3. 클래스에는 public, default만 사용 가능하다.
4. public 클래스는 파일명과 같아야 한다.
5. 캡슐화는 데이터를 보호하고 기능을 통해 접근하게 만든다.
