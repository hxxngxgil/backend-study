# Package & Import (패키지와 import)

## 패키지란
패키지는 관련된 클래스들을 모아서 관리하는 폴더 개념이다.  
코드를 구조적으로 정리하고 관리하기 위해 사용한다.

---

## 패키지 사용 이유
- 관련된 클래스들을 그룹화
- 코드 구조 정리
- 클래스 이름 충돌 방지

---

## import 사용 이유

다른 패키지에 있는 클래스를 사용할 때는 전체 경로를 입력해야 한다.

예시

com.company.myapp.Student student = new com.company.myapp.Student();

→ 너무 길고 불편함

그래서 import를 사용한다.

---

## import 사용 방법

import를 사용하면 클래스 경로를 생략할 수 있다.

예시

import com.company.myapp.Student;

Student student = new Student();

---

## 패키지 위치에 따른 사용

### 같은 패키지
같은 패키지에 있는 클래스는 import 없이 사용할 수 있다.

---

### 다른 패키지
다른 패키지에 있는 클래스는 전체 경로를 입력하거나 import를 사용해야 한다.

---

## 전체 import

패키지 안의 모든 클래스를 가져오고 싶으면 * 을 사용한다.

예시

import com.company.myapp.*;

---

## 클래스 이름이 중복될 때

같은 이름의 클래스가 여러 개 존재하면  
패키지명을 포함한 전체 경로로 구분할 수 있다.

이 경우 import는 하나만 선택해서 사용하는 것이 일반적이다.

또는 둘 다 import하지 않고 전체 경로를 직접 사용해도 된다.

---

## 패키지 규칙

### 1. 패키지 이름과 폴더 구조는 같아야 한다 (필수)

패키지 경로와 실제 폴더 구조는 반드시 동일해야 한다.

---

### 2. 도메인 역순 사용 (관례)

보통 회사 도메인을 거꾸로 사용한다.

예시

com.company.myapp

---

### 3. 소문자 사용 (관례)

패키지 이름은 모두 소문자로 작성한다.

---

### 4. 내부 프로젝트는 자유롭게 사용 가능

외부에 공개하지 않는 경우  
도메인 규칙을 꼭 따르지 않아도 된다.

(패키지 이름이 너무 길어질 수 있기 때문)

---

## 패키지 구성 방법

- 서로 관련된 클래스는 같은 패키지에 모은다.
- 관련이 적은 클래스는 다른 패키지로 분리한다.

→ 이렇게 하면 코드 구조를 이해하기 쉽고 유지보수가 편해진다.

---

## 핵심 정리

1. 패키지는 클래스를 정리하는 폴더 개념이다.
2. 다른 패키지 클래스는 import 또는 전체 경로가 필요하다.
3. import를 사용하면 코드가 간결해진다.
4. * 를 사용하면 패키지 전체를 import할 수 있다.
5. 패키지 이름은 폴더 구조와 반드시 일치해야 한다.
6. 관련된 클래스끼리 묶는 것이 중요하다.
